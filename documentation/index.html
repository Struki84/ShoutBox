About the project

When I first started doing web, one of the first thing I wanted to have on my web was a shoutbox. A basic feedback tool, that most people want on their websites. A place where your web visitors can post their impressions and comments, is, for most people, first piece of dynamic web content they come across. My expirience was no different, natural transition from html/css to my first lines of php and sql, started when I wanted to include this feature in some of my websites. At that time I didn't know much about php or sql, so building a shoutbox was a confusing and furstrating process for me.

The idea behind the project, is that you should be able to include this tool in your website with only 2 or 3 lines of code. This is possible trough something called objective programming, that is integrated into php5. Now for some people all of this is well known nad they'r saying: "what is this guy jibbering about, I know all of this", but my idea was to give this tool to people how don't know much about programming and have no intetion learning how to do it, but want a simple home page or an online photo album with some feedback from their visitors. It all started as a small exercise for myself in objective oriented PHP, in combination with my own need for this feature, I wrote this small framework so that I don't need te rewrite the same piece of code over and over again.

How to use

This should be simple. First, you need to get all the files needed, since you'r reading this there's a big chance you allready did that, but in case you didn't here is the link, just press the button which says ZIP. Inside the shoutbox folder there should be this documentation, some examples, and the heart of everything, the folder called mvc. The mvc folder is the one you HAVE TO PUT IN YOU'R WEB SITE FOLDER for the whole thing to work, it is essential. There is a css folder which takes care of your shoutbox look and feel, but you can always create your own css file, just make sure to link it properly, more on that later. The other really important thing is the shout_box_table.sql file. You will need this to create you'r database table where all the comments and the data about your visitors will be stored. If you don't know what I'm talking about, best advice would be to google about phpmyadmin, and how to create a database, then login into you'r phpmyadmin, usually by going to http://www.yourdomain.com/phpmyadmin create your database and then import the shout_box_table.sql file. If you never used phpmyadmin, don't freak out it's not that hard, basic point and click, just spend 20 mins googling and reading and you'll get it.

If you don't have much expirience with this things, follow my lead. In you'r root folder(thats the one with your index file) create a new folder called shoutbox. Inside that folder copy the folders mvc and css from the folder you just downloaded. Now, keep in mind one thing, you can include the shoutbox only in the files that have *.php extension, so before including your shoutbox change the extension of the file where you will do the insertion.

So if you followed my lead, you should have one file named, some_file.php, in that file first line of code you should put is:

require "shoutbox/mvc/controller.php"

This will include all the files needed for the shoutbox to work in you'r some_file.php, so if you want to include it in some other file you should put the same line on top of you'r some_other_file.php. Next, you have to start the engine, in this case it's called instancing an object, if you want to now more google it, if not just type:

$controller = new Controller('host', 'username', 'password', 'database_name', 'port');

This line will start all the logic needed to run the shoutbox and connect to your database by creating controller object. The variable $controller is called the handler, it handles your object Controller and you can call it anything you want, $shoutbox for example, again google if you are interested for more info. Of course you have to fill in the parameters for your database and change 'host' with you'r host (usually 'localhost'), 'username' with you'r database user and so on. So this line filled properly should look something like this

$controller = Controller('localhost', 'root', 'r00t', 'development')

This is an example for my database, you should do the same but acording to your database parameters. The port parameter is not set in my example, bacuse it's preset as the default MySQL port 3306 which in 99% of time won't change, but if it does, just put the changed one in, at the end of the line. Now there is a small warning I must issue, if by some chance you are using some other framework or/and some other class library you shoudn't have two handlers of the same name on the same site. Also, you could construct your web site so that you have to include the controller.php file only once, and so that you have to create the object Controller only once, but then you should look into some more php and how to include them into one strucutre. But in the meantime, you have to include the file and instaciate the object for each page you want your shoutbox in.

Thats it! Everything is set for using the shoutbox and the good news is, this was the hard part.

For the advanced users!

All the connections to the database and queries are handled in mvc/model.php, mvc/controller.php takes care of the communication user-data, and the form is in mvc/view.php.

Insert methods

The methods described here are what actually generates the shoutbox.

  • insertShoutBox('name', 'type', 'order');
  • insertInput('name', 'type');
  • insertOutput('name', 'type', 'order');

Inside there are parameters name, type and order.

name - The name of the shoutbox is assigned as a css id attribute which is used when creating style for the shoutbox

type - Will specify what the shoutbox is used for, so when getting the posts from the database (or putting them in), they will bi associated with the shoutbox of that type. If for eg. you whant to use it as a blog comment tool, you can use one css style nd pass the blog id for specific blog type.

order - Will set order you want the shout box to appear, by default its set to ascend. Meaning that the input form and the latest post will be on the bottom. Input 'desc', to reverse the order, meaning the form and the latest post will bo on top.

These methods are written inside the object we just created, and the handler is the thing that..., well handles them. So the way that you call the methods is trough the handler by typing:

$controller->insertShoutBox('name', 'type', 'order');

It will generate the entire shoutbox inside a div tag and assign the id nameShoutBox.

$controller->insertInput('name', 'type');

$controller->insertOutput('name', 'type', 'order');

This two methods are used when you want to seperate the input and the output and put them in different places on a page, or even within the website. Keep in mind that if you use input and output methods they need to have the same type.

Styling

This is simple, elements are all preset and the ids are half-generated. Look into the css file for more details. The logic is that when you enter a name for your shoutbox, it is automaticly appended to the elemnent id. So in the case where you input entire shoutbox you get the id for the entire shoutbox div specified by the name parameter.

#your_nameShoutBox{/*some css here*/}

You can create your own css code wherever you want the only thing to keep in mind is the name of the shoutbox you entered and append it to the preset id starting with the capital letter. Also if you use input and output separately you have to keep track of the name parameter when creating css:

#your_nameOutput{/*some css here*/}

#your_nameInput{/*some css here*/}
